## Architecture Highlights

This component demonstrates several architectural best practices:

### 1. API Design Excellence

**Foreign Key Pattern:**

- Uses `StringValueOrRef` for cross-resource references
- Enables declarative dependencies (e.g., `subnets` can reference `AwsVpc`)
- Supports both literal values and resource references

**80/20 Scoping:**

- Simple flat structure for common use cases
- Abstractions for DNS (`AwsAlbDns`) and SSL (`AwsAlbSsl`)
- Extensible for future advanced features (target groups, listener rules, WAF)

### 2. Validation Strategy

**Multi-AZ Enforcement:**

```protobuf
repeated org.project_planton.shared.foreignkey.v1.StringValueOrRef subnets = 1 [
  (buf.validate.field).required = true,
  (buf.validate.field).repeated.min_items = 2,
];
```

This prevents the most common ALB deployment anti-pattern (single-AZ deployment) at the API level.

### 3. Production Readiness

**Built-in Best Practices:**

- Deletion protection support
- Configurable idle timeout with recommended default (60s)
- SSL/TLS termination abstraction
- Route53 DNS integration
- Security group attachment

### 4. Multi-IaC Support

Both Pulumi and Terraform implementations follow identical patterns:

- Clean module structure
- Separation of concerns (ALB, DNS in separate files)
- Consistent naming conventions
- Proper output handling
