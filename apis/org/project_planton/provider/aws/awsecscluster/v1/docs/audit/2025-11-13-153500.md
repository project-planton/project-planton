## Critical Gaps

**⚠️ One Important Gap** - Not Blocking Production, But Limits Cost Optimization

### Missing: Default Capacity Provider Strategy Configuration

**Current State:**
- Component allows specifying capacity providers (`FARGATE`, `FARGATE_SPOT`)
- Terraform creates a basic default strategy (first provider, weight=1, base=0)
- Pulumi doesn't create any default strategy

**Problem:**
The excellent research documentation (docs/README.md) identifies the **base/weight strategy** as "the most important lever for Fargate cost optimization," providing up to 70% cost savings. However, the current API doesn't expose this critical configuration.

**Example from Research:**
> "A production pattern: `FARGATE (base: 1, weight: 1)` and `FARGATE_SPOT (weight: 4)` means that for every 5 new tasks scaled, one is on-demand and four are Spot."

**Impact:**
- Users cannot implement production-grade cost optimization patterns
- Terraform creates a suboptimal default (no base guarantee, equal weights)
- Research doc recommends a feature that the API doesn't support

**Recommendation:**
Add to `spec.proto`:

```protobuf
message AwsEcsClusterSpec {
  // ... existing fields ...
  
  // default_capacity_provider_strategy defines the base/weight
  // distribution for tasks across capacity providers. This is the
  // primary cost-optimization lever for Fargate workloads.
  repeated CapacityProviderStrategyItem default_capacity_provider_strategy = 4;
}

message CapacityProviderStrategyItem {
  // capacity_provider name (e.g., "FARGATE" or "FARGATE_SPOT")
  string capacity_provider = 1 [(buf.validate.field).required = true];
  
  // base is the minimum number of tasks to run on this provider
  int32 base = 2;
  
  // weight is the relative proportion for scaling tasks beyond the base
  int32 weight = 3 [(buf.validate.field).int32.gte = 0];
}
```

This is the only significant gap preventing this component from being 100% production-ready.
